Coursework Playlist

This page is for class projects where I practiced data analysis, programming, research, and technical communication. My main project in this category is my COGS 108 final project.

Track List

COGS 108: Analyzing and Forecasting U.S. Rental Price Trends

Problem: Rental prices in the United States have changed a lot over the past decade. Our project looked at how rents changed over time and whether past rental data could help forecast future rental prices.

Contribution: My role was analysis and visualization. I helped explore the rental data, create plots, compare trends across regions, and explain what the results showed.

Tools: Python, Jupyter Notebook, pandas, NumPy, matplotlib, Zillow rental datasets, time-series analysis, and forecasting methods.

Constraints: The data had missing values and did not cover every city equally. Some datasets also ended before the later COVID-era rent changes, so we had to be careful about what conclusions we could make.

Status: Completed as a group final project for COGS 108.

What I learned: This project helped me get better at working with real datasets. The data was not perfectly clean, and the national trend did not explain every city or metro area. I learned that visualizations are useful because they can show patterns that are easy to miss in a table.

CSE 29: Web Chat Server

Problem: This project was about building a small web-based chat server that could let multiple users send and receive messages in real time.

Contribution: I worked with C socket programming to handle server logic, HTTP requests, and WebSocket communication between clients and the server.

Tools: C, HTTP, WebSockets, socket programming, and command-line testing.

Constraints: Since the server was written in C, I had to be careful with lower-level details like connections, buffers, request handling, and how data moved between the client and server.

Status: Completed as a CSE 29 coursework project.

What I learned: This project helped me understand what happens underneath higher-level web frameworks. It made HTTP and real-time communication feel less abstract because I had to work closer to the server logic myself.
